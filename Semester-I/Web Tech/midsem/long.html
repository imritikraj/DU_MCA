HTML Fundamentals

What is HTML and how does it differ from CSS and JavaScript?
HTML HTML (HyperText Markup Language) is the standard markup language for creating web pages and web applications. It provides the structural foundation of web content.
Core Differences:
Technology Purpose Role
HTML Structure & Content Defines page structure (headings, paragraphs, images)
CSS Presentation & Styling Controls appearance (colors, layout, fonts)
JavaScript Behavior & Interactivity Adds dynamic functionality (animations, form validation)
<!-- HTML: Structure -->
<button id="myBtn">Click Me</button>

<!-- CSS: Styling -->
<style>
#myBtn {
  background: blue;
  color: white;
  padding: 10px 20px;
}
</style>

<!-- JavaScript: Behavior -->
<script>
document.getElementById('myBtn').addEventListener('click', function() {
  alert('Button clicked!');
});
</script>
Think of HTML as the skeleton, CSS as the skin/clothes, and JavaScript as the muscles that make everything move!
What are HTML5 Semantic Elements and why are they important?
HTML5 Semantic elements clearly describe their meaning to both the browser and the developer, making code more readable and maintainable. Key Semantic Elements:
  • <header> - Introductory content or navigation
  • <nav> - Navigation links
  • <main> - Main content area (should be unique)
  • <article> - Self-contained, reusable content
  • <section> - Thematic grouping of content
  • <aside> - Sidebar or tangential content
  • <footer> - Footer content
  • <figure> - Self-contained flow content
  • <figcaption> - Caption for figure
<body>
  <header>
    <nav>...</nav>
  </header>
  <main>
    <article>
      <section>...</section>
    </article>
    <aside>...</aside>
  </main>
  <footer>...</footer>
</body>
Benefits:
  • Better SEO: Search engines understand content structure
  • Improved Accessibility: Screen readers can navigate better
  • Cleaner Code: More readable and maintainable
  • Future-Proof: Better compatibility with new technologies
Use semantic elements whenever possible. They make your code self-documenting and improve both SEO and accessibility!
What is the purpose of the DOCTYPE declaration?
DOCTYPE The DOCTYPE declaration is an instruction to the web browser about what version of HTML the page is written in. It's the first line in an HTML document and triggers standards mode rendering.
<!DOCTYPE html> <!-- HTML5 DOCTYPE -->
<html>
<head>
  <title>My Page</title>
</head>
<body>
  <!-- Content -->
</body>
</html>
Why DOCTYPE is Essential:
  • Prevents Quirks Mode: Older browsers had inconsistent rendering. DOCTYPE ensures standards mode
  • Consistent Rendering: Ensures pages look the same across different browsers
  • HTML Validation: Required for valid HTML documents
  • Future Compatibility: Ensures compatibility with new browser features
Without a proper DOCTYPE, browsers may render your page in "quirks mode" which can lead to unexpected layout issues and inconsistent styling!
Historical DOCTYPE Examples:
<!-- HTML 4.01 Strict -->
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">

<!-- XHTML 1.0 Transitional -->
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
HTML5 simplified the DOCTYPE to just <!DOCTYPE html>. It's case-insensitive but lowercase is the convention.
What's the difference between ID and Class attributes?
Attributes
Feature ID Attribute Class Attribute
Uniqueness Must be unique in document Can be reused multiple times
CSS Selector #id-name .class-name
JavaScript getElementById() getElementsByClassName()
Specificity High (100 points) Medium (10 points)
Use Case Unique elements, page anchors Styling groups of elements
<!-- ID: Unique identifier -->
<div id="main-header">Welcome</div>

<!-- Class: Reusable styling -->
<button class="btn btn-primary">Submit</button>
<button class="btn btn-secondary">Cancel</button>

<!-- CSS -->
<style>
/* ID selector */
#main-header {
  color: #333;
  font-size: 2rem;
}
/* Class selector */
.btn {
  padding: 10px 20px;
  border: none;
  border-radius: 4px;
}
.btn-primary { background: blue; color: white; }
.btn-secondary { background: gray; color: black; }
</style>
Best Practices:
  • Use IDs for unique elements that need JavaScript targeting or page anchors
  • Use classes for styling and grouping similar elements
  • Avoid using IDs for CSS styling (high specificity can cause issues)
  • Elements can have both ID and multiple classes
When naming IDs and classes, use kebab-case (hyphens) like main-header rather than camelCase for better readability and consistency.

How do HTML Forms work and what are common input types?
Forms HTML Forms are used to collect user input and send it to a server for processing. They consist of form elements like input fields, checkboxes, radio buttons, and submit buttons.
<form action="/submit-data" method="POST" enctype="multipart/form-data">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required>

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required>

  <button type="submit">Submit</button>
</form>
Common HTML5 Input Types:
  • text - Single-line text input
  • password - Masked text input
  • email - Email with validation
  • url - URL with validation
  • tel - Telephone number
  • number - Numeric input
  • range - Slider control
  • date - Date picker
  • time - Time picker
  • color - Color picker
  • file - File upload
  • checkbox - Multiple selection
  • radio - Single selection
  • submit - Form submission
  • reset - Form reset
Form Attributes:
  • action - URL where form data is sent
  • method - HTTP method (GET/POST)
  • enctype - Encoding type for file uploads
  • target - Where to display response
  • novalidate - Disable browser validation
Always use the <label> element with the for attribute pointing to the input's id. This improves accessibility and allows clicking the label to focus the input!
What is the purpose of the <meta> tag and common attributes?
Meta Tags The <meta> tag provides metadata about the HTML document. This information is not displayed on the page but is used by browsers, search engines, and other web services. Common Meta Tags:
<!-- Character encoding -->
<meta charset="UTF-8">

<!-- Viewport for responsive design -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

<!-- Page description for SEO -->
<meta name="description" content="Free web development tutorials">

<!-- Keywords (less important now) -->
<meta name="keywords" content="HTML, CSS, JavaScript">

<!-- Author -->
<meta name="author" content="John Doe">

<!-- Refresh/redirect -->
<meta http-equiv="refresh" content="5; url=https://example.com">

<!-- Open Graph for social media -->
<meta property="og:title" content="My Website">
<meta property="og:description" content="Website description">
<meta property="og:image" content="image.jpg">
Essential Meta Tags:
  • charset="UTF-8" - Ensures proper character encoding
  • viewport - Critical for responsive mobile design
  • description - Important for SEO and social sharing
The viewport meta tag <meta name="viewport" content="width=device-width, initial-scale=1.0"> is absolutely essential for responsive web design. Without it, mobile devices may render your page at desktop scale!
Place the charset meta tag first in the <head> section to ensure the browser knows the character encoding before parsing any text content.
What is the alt attribute and why is it important for images?
Accessibility SEO The alt attribute (alternative text) provides a textual description of an image. It's essential for accessibility, SEO, and when images fail to load.
<!-- Descriptive alt text for meaningful images -->
<img src="company-logo.png" alt="Acme Corporation Logo">

<!-- Empty alt for decorative images -->
<img src="decorative-border.png" alt="">

<!-- Functional image (button) -->
<button>
  <img src="search-icon.png" alt="Search">
</button>
Why alt attribute is crucial:
  • Accessibility: Screen readers read alt text to visually impaired users
  • SEO: Search engines use alt text to understand image content
  • Fallback: Displayed when images fail to load
  • Performance: Helps with image search rankings
Writing Effective Alt Text:
  • Descriptive images: Describe what the image shows
  • Functional images: Describe the action (e.g., "Search")
  • Decorative images: Use empty alt attribute (alt="")
  • Complex images: Provide a brief description
When writing alt text, imagine you're describing the image to someone over the phone. Be concise but descriptive enough to convey the same information the image provides.
Never omit the alt attribute entirely. If an image is decorative, use alt="" (empty string) instead of no alt attribute. This tells screen readers to skip the image entirely.

CSS Fundamentals

What is the CSS Box Model and how does it work?
Box Model The CSS Box Model describes how every HTML element is represented as a rectangular box with specific properties that determine its size, spacing, and layout.
<style>
.box {
  /* Content dimensions */
  width: 200px;
  height: 100px;
  
  /* Padding: space inside */
  padding: 20px;
  
  /* Border: line around */
  border: 5px solid black;
  
  /* Margin: space outside */
  margin: 30px;
  
  /* Background shows in padding */
  background: lightblue;
}
</style>

<div class="box">Content</div>
Box Model Components:
  • Content: Actual text/images (innermost area)
  • Padding: Space between content and border (background extends here)
  • Border: Line surrounding padding and content
  • Margin: Space between elements (transparent, no background)
Box-Sizing Property:
<style>
/* content-box (default) */
.content-box {
  box-sizing: content-box;
  width: 200px;
  padding: 20px;
  border: 5px solid black;
  /* Total width: 200 + 20 + 20 + 5 + 5 = 250px */
}

/* border-box (recommended) */
.border-box {
  box-sizing: border-box;
  width: 200px;
  padding: 20px;
  border: 5px solid black;
  /* Total width: 200px (content shrinks to 150px) */
}
</style>
Always use box-sizing: border-box for predictable layouts. You can apply it globally: * { box-sizing: border-box; }
Without box-sizing: border-box, adding padding or border to an element increases its total size, which can break layouts and cause unexpected behavior!

Practical Examples:
<!-- Inline: For text styling -->
<p>
  This text has <span style="color: red;">red words</span> inside it.
</p>

<!-- Block: For layout containers -->
<div style="background: #f0f0f0; padding: 20px;">
  <p>This is a paragraph inside a div container.</p>
</div>

<!-- Inline-block: For custom buttons -->
<button style="display: inline-block; width: 120px; height: 40px;">
  Custom Button
</button>
Use display: inline-block when you need elements to flow like inline elements but accept dimensions like block elements. Perfect for navigation menus, button groups, and custom form elements!
