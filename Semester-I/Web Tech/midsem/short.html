HTML Fundamentals

What is HTML & difference from CSS/JS?
HTML: Structure (skeleton)
CSS: Presentation (styling)
JavaScript: Behavior (interactivity)
HTML5 Semantic Elements
<header>, <nav>, <main>, <article>,
<section>, <aside>, <footer>
Provide meaning and structure
DOCTYPE Declaration Purpose
Triggers standards mode in browsers
Prevents quirks mode
Essential for consistent rendering
ID vs Class Attributes
ID: Unique identifier (#selector)
Class: Reusable group (.selector)
ID has higher CSS specificity
Div vs Span Elements
Div: Block-level (new line)
Span: Inline-level (flows with text)
Div for layout, Span for text styling
HTML Forms & Input Types
text, password, email, number, date
radio, checkbox, file, submit
Collects user input for server processing

CSS Fundamentals

CSS Box Model
Content â†’ Padding â†’ Border â†’ Margin
box-sizing: border-box includes padding+border in width/height
Display: Inline vs Block
Inline: Flows with text, no width/height
Block: New line, accepts width/height
Inline-block: Hybrid of both
CSS Position Property
static: Normal flow (default)
relative: Offset from normal position
absolute: Relative to positioned parent
fixed: Relative to viewport
Margin vs Padding
Margin: Space between elements (outside)
Padding: Space inside element (around content)
Margin collapses, Padding doesn't
z-index Property
Controls stacking order (depth)
Higher value = closer to user
Requires positioned element
CSS Specificity
Inline > ID > Class > Element
Determines which styles apply when conflicts occur

Advanced Concepts

CSS Grid vs Flexbox
Grid: 2D layout (rows + columns)
Flexbox: 1D layout (row OR column)
Use Grid for overall layout, Flexbox for components
Media Queries
@media (min-width: 768px) { ... }
Adapt styles based on screen size
Essential for responsive design
CSS Preprocessors (SASS)
Variables, nesting, mixins, functions
Better organization and maintainability
Compiles to regular CSS
Cross-Browser Compatibility
Use CSS resets/normalize
Vendor prefixes (-webkit-, -moz-)
Feature detection with @supports
CSS Performance Optimization
Minify CSS, Remove unused code
Critical CSS inlining
Efficient selectors, GPU animations
Web Accessibility
Semantic HTML, ARIA attributes
Color contrast, Keyboard navigation
Screen reader compatibility

Quick Reference

CSS Units
px: Absolute pixels
%: Relative to parent
em: Relative to font-size
rem: Relative to root font-size
vw/vh: Viewport dimensions
Common Breakpoints
Mobile: < 768px
Tablet: 768px - 1024px
Desktop: > 1024px
Use mobile-first approach
CSS Selectors
.class, #id, element
[attribute], :pseudo-class
::pseudo-element, > child
+ adjacent, ~ general sibling
